# MeshComm

> **인터넷·기지국 없이 동작하는 P2P BLE 오프라인 메신저**  
> 재난·통신 마비 상황에서 스마트폰만으로 주변 사람들과 연결됩니다.

---

## 이런 상황에서 사용합니다

- 홍수·지진으로 기지국이 마비된 재난 현장
- 지하·산악 등 통신 음영 지역
- 인터넷 없는 오프라인 환경에서의 소규모 통신

```
[폰 A] ←BLE→ [폰 B] ←BLE→ [PC] ←BLE→ [폰 C]
              ↑ 릴레이          ↑ 릴레이
        A와 C가 직접 연결되지 않아도 B·PC가 메시지를 중계합니다
```

각 기기가 동시에 서버 역할을 하며, 연결 범위를 이어붙여 통신 거리를 확장합니다.

---

## 지원 플랫폼

| 플랫폼 | 역할 |
|--------|------|
| Android | 메시지 송수신 + 메시 릴레이 (백그라운드 동작) |
| Windows PC | 항상 켜진 안정 릴레이 + 관리용 UI |

> **PC 연결 주의:** Windows PC는 연결할 Android 기기를 미리 Windows 설정 → Bluetooth에서 **수동 페어링**해야 합니다. 폰↔폰 연결은 페어링 없이 즉시 연결됩니다.

---

## 처음 시작하기

### 1. 앱 실행 후 신원(ID) 생성
앱을 처음 실행하면 자동으로 기기 고유 ID와 암호화 키가 생성됩니다.  
Settings 화면에서 내 이름을 설정합니다.

### 2. 상대방과 QR 코드로 연결
연락처를 추가하려면 **QR 코드로 공개키를 교환**합니다.

1. 하단 **QR** 탭 → 내 QR 코드 표시
2. 상대방이 **SCAN** 탭에서 QR 버튼으로 스캔
3. 연결 확인 후 연락처 등록 완료

> BLE 범위 내에 있으면 자동으로 기기를 발견합니다. QR 교환 없이도 연결은 되지만, 공개키 진위 확인(TOFU)을 위해 QR 대조를 권장합니다.

### 3. Android 위치 권한 허용
BLE 스캔에는 **위치 권한**과 **위치 서비스(GPS) 활성화**가 필요합니다 (삼성 기기 기준).  
앱 첫 실행 시 권한 요청이 표시됩니다.

## 화면 구성

### 홈 화면 — 왼쪽 탭

| 탭 | 내용 |
|----|------|
| 공지 | 수신된 공지 메시지 목록 |
| **All** | 전체 연락처 목록 |
| **Group** | 그룹 채팅 목록 |
| 즐겨찾기 | 즐겨찾기 등록된 연락처 |
| 채팅 | 대화 기록이 있는 연락처만 표시 |
| Import | 연락처 JSON 가져오기 |
| Export | 연락처 JSON 내보내기 |

> **미읽음 표시:** All·Group 탭에 미읽음 메시지가 있으면 아이콘 오른쪽 위에 **주황색 점이 깜박입니다**.

## 1:1 채팅

연락처를 탭하면 채팅 화면이 열립니다.

### 메시지 모드

우측 하단 드롭다운으로 전송 모드를 선택합니다:

| 모드 | 설명 |
|------|------|
| **일반** | 기본 채팅 메시지 |
| **타임** | 수신 후 **1분 뒤 자동 삭제** — 민감한 내용 공유 시 |
| **파일** | 파일 선택 후 전송 |
| **이미지** | 이미지 선택 후 전송 |

### 이미지 뷰어
- 이미지 탭 → 전체화면 뷰어
- 오른쪽 상단 버튼: **저장** / **삭제** / **닫기**
- 저장 시 파일 탐색기에서 저장 위치를 선택합니다

### 파일 저장
수신한 파일 탭 → 파일 탐색기에서 저장 위치 선택

---

## 그룹 채팅

### 그룹 만들기
1. **Group** 탭 하단 **+ 그룹 만들기** 버튼
2. 그룹 이름 입력 → 초대할 연락처 선택 → 확인
3. 선택한 연락처에 초대장이 자동 전송됩니다

### 그룹 초대 수락
홈 화면 상단에 초대 알림이 표시됩니다. **수락** 또는 **거절** 선택.

### 그룹 채팅 화면

연락처 채팅과 동일한 모드를 지원합니다:

| 모드 | 설명 |
|------|------|
| **일반** | 그룹 메시지 |
| **타임** | 수신 후 1분 자동 삭제 |
| **파일** | 파일 전송 |
| **이미지** | 이미지 전송 |

### 그룹 관리 (방장 전용)

채팅 화면 우측 상단 **⋮** 메뉴:

| 메뉴 | 설명 |
|------|------|
| 멤버 추방 | 선택한 멤버를 그룹에서 제거. 해당 멤버 기기에 "추방되었습니다" 알림 표시 |
| 그룹 이름 변경 | 내 화면의 표시 이름만 변경 (다른 멤버에게 전파되지 않음) |

### 그룹 나가기
- **⋮** 메뉴 → **그룹 나가기**
- 방장이 나갈 경우 다른 멤버에게 방장이 자동 이전됩니다
- 마지막 멤버가 나가면 그룹이 삭제됩니다

### 그룹 백업 / 복원
- Group 탭 하단 **Backup** → 그룹 목록을 JSON 파일로 저장
- **Restore** → 백업 파일에서 그룹 복원

---

## SCAN — 메시 네트워크 시각화

하단 **SCAN** 탭에서 현재 연결된 메시 구조를 확인합니다.

- **SCAN START** 버튼 → 주변 노드를 N-depth까지 탐색
- 각 원에 노드 이름·홉 수 표시
- 직접 연결된 이웃(1홉)과 릴레이를 통한 원거리 노드를 한눈에 파악
- depth 값으로 탐색 범위 조절 가능

---

## QR 코드

하단 **QR** 탭:

- **내 QR 코드 표시** — 상대방이 스캔해서 나를 연락처에 추가
- **QR 스캔** — 상대방 QR을 스캔해서 연락처 추가 + 공개키 진위 확인

## Settings

우측 상단 **⚙** 버튼:

| 항목 | 설명 |
|------|------|
| 내 이름 | 상대방에게 표시될 이름 |
| 역할 | Creator / Builder / Admin / User — 역할에 따라 공지 쿨다운·연락처 한도가 다름 |
| Identity Backup | 내 ID(키쌍)를 비밀번호로 암호화해서 파일로 저장 |
| Identity Restore | 백업 파일에서 ID 복원 (재설치 후 동일 ID 사용 가능) |
| 데모 모드 | 가상 27노드 메시 시나리오로 UI 확인 |

> **Identity Backup 권장:** 기기를 교체하거나 앱을 재설치하면 ID가 바뀝니다. 미리 백업해두면 기존 연락처와 동일한 신원으로 복귀할 수 있습니다.

---

## 연락처 관리

연락처 항목에서 **⋮** 메뉴:

| 메뉴 | 설명 |
|------|------|
| 즐겨찾기 | 즐겨찾기 탭에 상단 고정 |
| 역할 변경 | 해당 연락처의 역할 설정 (자신보다 낮은 역할만 변경 가능) |
| 그룹 채팅에 초대 | 기존 그룹에 이 연락처 초대 |
| 삭제 | 연락처 삭제 (채팅 기록 유지) |

## 보안

| 항목 | 방식 |
|------|------|
| 기기 ID | Ed25519 공개키 기반 — 위조 불가 |
| 메시지 암호화 | X25519 ECDH + AES-GCM — 릴레이 노드도 내용 열람 불가 |
| 패킷 서명 | Ed25519 — 발신자 위조 방지, 검증 실패 시 즉시 폐기 |
| 공개키 확인 | QR 핑거프린트 대조 (TOFU) |
| Identity 백업 | PBKDF2 + AES-GCM 암호화 — 앱은 비밀번호를 저장하지 않음 |

## 알아두면 좋은 점

- **BLE 범위**: 약 10–50m (환경에 따라 다름). 사람이 많을수록 릴레이로 범위가 확장됩니다.
- **백그라운드 동작 (Android)**: 앱을 닫아도 메시 릴레이가 백그라운드에서 계속 동작합니다.
- **타임 메시지**: 보내는 쪽과 받는 쪽 모두 1분 후 자동 삭제됩니다.
- **파일 저장 위치**: Android는 `Documents/Mesh-comm/`, Windows는 `내 문서\Mesh-comm\` 아래에 자동 정리됩니다.
- **LAN 전송**: 같은 Wi-Fi 네트워크에 있으면 BLE 대신 LAN으로 자동 전환되어 더 빠르게 전송됩니다.
